# Workflow

## 7D Methodology
1. **Define** — Capture requirements in `product.md` and track specs.
2. **Design** — Architecture, folder structure, module decomposition in implementation plan.
3. **Develop** — Write code per approved plan.
4. **Debug** — Test with dry-run mode, validate on both OS targets.
5. **Document** — README, inline docstrings, walkthrough.
6. **Deliver** — Tag release, generate launchers (.bat/.sh).
7. **Deploy** — Push to GitHub, install on target machines.

## Process Rules
- No code changes without an approved implementation plan.
- All modifications start as dry-run diffs.
- State must be valid before any turn begins.
